# 日报生成器

从 git 提交记录中自动提取工作内容，结合用户补充，生成标准化日报保存到 `docs/` 目录。

## 第一步：提取今日 git 提交

运行以下命令获取今日所有提交：

```bash
git log --since="00:00:00" --until="23:59:59" --pretty=format:"%h %s" --no-merges
```

**整理规则：**
- 去掉 commit hash，只保留描述
- 将 `feat:` / `fix:` / `refactor:` / `chore:` 等前缀转为自然语言（例：`feat: 添加登录页` → `添加登录页功能`）
- 将明显属于同一功能的多次小提交合并为一条
- 单独提取所有含 `fix` / `bug` / `bugfix` / `hotfix` 关键词的提交，用于推断「遇到的问题」

如果当前目录不是 git 仓库，或今日没有任何提交，直接告知用户并请他口述今日完成内容。

## 第二步：询问补充信息

展示提取到的今日完成内容和推断出的问题，然后一次性问清楚：

> 我从 git 记录里提取了今天的工作内容，请帮我确认/补充：
>
> **遇到的问题**：[列出从 fix/bug commit 推断的内容，如果没有则写"暂未发现"]，有需要补充或修改的吗？
>
> **明日计划**：明天打算做什么？

等用户回复后再生成报告，不要跳过这一步。

## 第三步：生成并保存报告

使用以下模板生成报告，日期填写今天的实际日期：

```markdown
# 工作日报 - YYYY-MM-DD

## 今日完成

- [整理后的完成内容，每条一行]

## 遇到的问题

- [推断 + 用户补充的内容；如果真的没有，写"无"]

## 明日计划

- [用户提供的计划，每条一行]
```

**保存：**
- 路径：`docs/YYYY-MM-DD.md`（例：`docs/2026-03-24.md`）
- 如果 `docs/` 目录不存在，先创建它
- 如果当天文件已存在，询问用户是否覆盖

## 第四步：完成确认

保存后在对话中显示报告全文，并告知保存路径：

> 日报已保存到 `docs/2026-03-24.md` ✓
